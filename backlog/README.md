# Backlog 디렉토리

이 디렉토리는 제품 백로그 및 스프린트별 백로그를 저장합니다.

파일 규칙:
- 제품 백로그: `PRODUCT_BACKLOG.md` (한 파일)
- 스프린트 백로그: `sprint-<번호>-backlog.md` (예: `sprint-1-backlog.md`)
- 각 스프린트 백로그는 커밋된 스토리 목록, 목표, 스프린트 기간을 포함해야 합니다.

예시:
- `PRODUCT_BACKLOG.md` — 제품 전체 백로그
- `sprint-1-backlog.md` — 첫 번째 스프린트에 할당된 항목

관리 지침:
- 백로그 항목을 업데이트할 때는 변경 사유를 헤더에 기록하세요.
- 큰 스토리는 스프린트 커밋 전 작은 작업으로 분해하세요.
